# 📊 MongoDB Atlas Setup - Step by Step

Complete guide to set up MongoDB Atlas for Swift Clinic Pro

---

## ⏱️ Time Required: 5 minutes

---

## 📋 Overview

We'll do these steps:
1. Create MongoDB Atlas account
2. Create a free cluster
3. Create database user
4. Get connection string
5. Allow network access
6. Test connection

**End result**: A connection string to use in Railway backend

---

## ✅ STEP 1: Create MongoDB Atlas Account

### 1a. Go to MongoDB Atlas
```
Open browser and go to:
https://www.mongodb.com/cloud/atlas
```

### 1b. Click "Sign Up"
- Look for green "Start Free" or "Sign Up" button
- Click it

### 1c. Choose How to Sign Up
You have 3 options:
- Sign up with **Google** (easiest)
- Sign up with **GitHub** (easiest)
- Sign up with **Email** (requires verification)

**Pick one and continue** ✓

### 1d. Enter Your Details
If using Email:
- Email: `your-email@example.com`
- Password: Create strong password
- Confirm password
- Check "I agree to the terms"
- Click "Create account"

### 1e. Verify Email (if using email signup)
- Check your email inbox
- Click verification link
- Your account is ready!

---

## ✅ STEP 2: Create Your First Cluster

### 2a. After Login - Choose Deployment Option
You'll see a screen asking:
```
"What would you like to do?"
```

Click: **"Build my own"** (or "Create a cluster")

### 2b. Select Free Tier
```
Tier Options appear:
- M0 (Free)      ← CLICK THIS
- M2
- M5
- etc.
```

**Click "M0 Sandbox"** (Free tier)

### 2c. Choose Provider & Region
```
Cloud Provider: AWS (or your preference)
Region: 
  - Pick one closest to you
  - Example: us-east-1, eu-west-1, ap-south-1
  - Default is fine
```

**Recommendation**: Pick a region close to your users

### 2d. Cluster Name
```
Default name: "Cluster0"
You can keep this or change to "medibook"
```

**Keep default or change to "medibook"** ✓

### 2e. Click "Create Deployment"
- Wait 2-3 minutes for cluster creation
- You'll see: "Cluster is being created..."
- Then: Green checkmark when done ✓

**Cluster is now ready!**

## ✅ STEP 3: Create Database User

### 3a. After Cluster is Created
You'll see your cluster dashboard with options:
```
- Connect
- Overview
- Metrics
- etc.
```

### 3b. Go to "Database Access"
On left sidebar, find:
```
SECURITY section
↓
Database Access  ← CLICK THIS
```

### 3c. Click "Add New Database User"
```
Green button: "+ Add New Database User"
Click it
```

### 3d. Enter User Credentials

**Method**: Choose "Autogenerated secure password"

Fill in:
```
Username: medibook

Password: [Auto-generated]
(Copy this - you need it later!)

Password Confirmation: [Auto-generated will auto-fill]
```

✅ **SAVE THIS PASSWORD SOMEWHERE SAFE!** (You'll need it soon)

### 3e. Set User Privileges
```
Database User Privileges:
- Built-in Role: "Atlas admin"   ← SELECT THIS
  (This gives full access for development)
```

### 3f. Click "Add User"
- Wait for user creation (a few seconds)
- You'll see: "Username: medibook" in the user list ✓

## ✅ STEP 4: Allow Network Access

### 4a. Go to "Network Access"
On left sidebar:
```
SECURITY section
↓
Network Access  ← CLICK THIS
```

### 4b. Click "Add IP Address"
```
Green button: "+ Add IP Address"
Click it
```

### 4c. Choose Network Access Option
```
Modal appears with options:
1. "Allow Access from Anywhere"    ← CLICK THIS
   (Allow: 0.0.0.0/0)
   
2. "Add Current IP Address"
   (Only your current IP)
   
3. Custom IP
```

### 4d. Select "Allow Access from Anywhere"
```
Whitelist Entry: 0.0.0.0/0
Comment: (optional) "Development"
```

### 4e. Click "Confirm"
- Wait for update (a few seconds)
- You'll see green checkmark ✓

## ✅ STEP 5: Get Connection String

### 5a. Go to "Clusters"
On left sidebar:
```
DEPLOYMENT section
↓
Clusters  ← CLICK THIS
```

### 5b. Click "Connect"
On your cluster:
```
Your Cluster0 card
↓
Green "Connect" button
Click it
```

### 5c. Choose "Connect your application"
```
Modal shows 3 options:
1. "Connect with MongoDB Shell"
2. "Drivers" (for applications)     ← CLICK THIS
3. "Connect via MongoSH"
```

### 5d. Select Driver & Version
```
Language: Node.js
Driver Version: (Keep default)
```

### 5e. Copy Connection String
```
You'll see a connection string:
mongodb+srv://medibook:<PASSWORD>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority

Steps:
1. Select all the text
2. Copy it (Ctrl+C)
```

### 5f. Replace PASSWORD
```
In the connection string:

FROM:
mongodb+srv://medibook:<PASSWORD>@cluster0.xxxxx...

TO:
mongodb+srv://medibook:YOUR_ACTUAL_PASSWORD@cluster0.xxxxx...

Replace: <PASSWORD> with the password you created earlier
```

**Keep this connection string safe!** You'll use it soon. ✓

## 📋 MongoDB Connection String Format

Your final connection string should look like:

```
mongodb+srv://medibook:YourPasswordHere@cluster0.abc123.mongodb.net/medibook?retryWrites=true&w=majority
```

Breaking it down:
```
mongodb+srv://          → Protocol (MongoDB Atlas)
medibook                → Username
:YourPassword           → Password (replace <PASSWORD>)
@cluster0.abc123...     → Cluster details
/medibook               → Database name
?retryWrites=true...    → Options
```

## ❌ Common Issues & Fixes

### Issue: "Password doesn't work"
**Fix**: Make sure you replaced `<PASSWORD>` with your actual password
```
Wrong: mongodb+srv://medibook:<PASSWORD>@...
Right: mongodb+srv://medibook:MyP@ssw0rd@...
```

### Issue: "Connection timeout"
**Fix**: Check network access is set to 0.0.0.0/0
1. Go to Network Access
2. Make sure entry shows: 0.0.0.0/0
3. If not, delete and add again

### Issue: "Authentication failed"
**Fix**: Check username and password
1. Username should be: `medibook`
2. Password should match what you set
3. Go to Database Access to verify or reset

### Issue: "Can't find Cluster0"
**Fix**: 
1. Go to Clusters section
2. Wait 2-3 minutes for cluster to finish creating
3. Refresh page (F5)

### Issue: Can't find the connection string
**Fix**:
1. Click on your Cluster0
2. Click green "Connect" button
3. Choose "Drivers"
4. Copy the string shown
